Detailed Description

Implementations should convert a libs.sprayfire.request.Uri object into a libs.sprayfire.request.RoutedUri object, using values from the libs.sprayfire.config.Configuration object that is passed in the constructor as defaults if needed.

Implementations of this interface should not necessarily be responsible for the actual instantiation and invocation of the necessary controller objects. That responsibility is left to a different object that is better suited for the task. Implementing classes should simply provide a RoutedUri object that can be used by the actual dispatching class.

See also:
https://github.com/cspray/SprayFire/wiki/Routing
